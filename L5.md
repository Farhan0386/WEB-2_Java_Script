## 1. Introduction to Event Handling

Event handling in JavaScript allows a web page to respond to user actions such as clicking, typing, scrolling, or hovering.

### Real-Life Analogy

- You do not open the door randomly.
- You wait for someone to knock.
- **Knock = Event**
- **Opening the door = Response Function**
- **Listener (You) = JavaScript**

JavaScript listens for events and executes code only when the event occurs.

---

## 2. Event Listener

An **event listener** is a mechanism that makes a web page react to user interactions.

### Key Points

- Listens for a specific event on an HTML element.
- Executes a function when the event occurs.
- Common events include:
  - click
  - keypress
  - hover
  - scroll

---

## 3. addEventListener()

`addEventListener()` is a built-in JavaScript method used to attach an event handler to an element.

### Syntax

```javascript
element.addEventListener("eventType", callbackFunction);
```

### Parameters

 **element** – HTML element to attach the event
 **eventType** – Type of event (e.g., `"click"`, `"keydown"`, `"submit"`)
 **callbackFunction** – Function executed when the event occurs

### Example

```html
<button id="btn">Click Me</button>
```

```javascript
document.getElementById("btn").addEventListener("click", function () {
  alert("Button was clicked!");
});
```

**Output:**
An alert box appears when the button is clicked.

---

## 4. removeEventListener()

`removeEventListener()` removes a previously attached event listener.

### Why it is Used

 Improves performance
 Better memory management
 Prevents unnecessary event execution

### Important Rule

The same function reference used in `addEventListener()` must be used to remove it.

# Syntax

```javascript
element.removeEventListener("eventType", eventHandler);
```

### Example

```html
<button id="btn">Click Me</button>
<button id="stop">Stop</button>
```

```javascript
const button = document.getElementById("btn");
const stopBtn = document.getElementById("stop");

function showMessage() {
  alert("Button clicked!");
}

button.addEventListener("click", showMessage);

stopBtn.addEventListener("click", function () {
  button.removeEventListener("click", showMessage);
});
```

**Result:**
After clicking **Stop**, the button will no longer respond.

## 5. Event Object

The **Event Object** is automatically passed to the event handler when an event occurs.

### Information Provided

* Type of event
* Target element
* Mouse position
* Keyboard key pressed
* Default behavior

### Example

```javascript
element.addEventListener("click", function (event) {
  console.log(event);
});
```

---

## 6. Common Properties of Event Object

| Property                  | Description                          |
| ------------------------- | ------------------------------------ |
| `event.type`              | Type of event (click, keyup, submit) |
| `event.target`            | Element that triggered the event     |
| `event.currentTarget`     | Element where listener is attached   |
| `event.clientX`           | Mouse X position                     |
| `event.clientY`           | Mouse Y position                     |
| `event.key`               | Key pressed (keyboard events)        |
| `event.preventDefault()`  | Prevents default browser behavior    |
| `event.stopPropagation()` | Stops event bubbling                 |

## 7. Concept Check (Exam-Oriented)

**Question:**
What are the two most essential arguments passed to `addEventListener()`?

**Correct Answer:**
Event type (e.g., `'click'`) and the listener (callback) function.

**Explanation:**

* Event type defines *what to listen for*
* Callback function defines *what action to perform*

---

## 8. Key Takeaways

* JavaScript reacts to user actions using events.
* `addEventListener()` attaches event handlers.
* `removeEventListener()` removes event handlers.
* Event Object provides detailed information about events.
* Callback functions define response logic.
